# Política de seguridad

## Versiones soportadas

El proyecto está en desarrollo activo (pre-1.0). Se da soporte de seguridad a la **última versión**
publicada y a la rama `main`.

## Reportar una vulnerabilidad

**No abras un issue público** para vulnerabilidades de seguridad.

Usá el reporte privado de GitHub:

1. Andá a la pestaña **Security** del repositorio.
2. **Report a vulnerability** (Private vulnerability reporting).

Incluí: descripción, pasos para reproducir, impacto estimado y, si podés, una propuesta de mitigación.

Procuramos responder dentro de **7 días**. Si confirmamos la vulnerabilidad, coordinamos un arreglo y
una divulgación responsable antes de hacerla pública.

Gracias por ayudar a mantener el proyecto seguro.
